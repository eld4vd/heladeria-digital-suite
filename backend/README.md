## Description

[Nest](https://github.com/nestjs/nest) framework TypeScript starter repository.

## Project setup

```bash
$ npm install
```

## Compile and run the project

```bash
# development
$ npm run start

# watch mode
$ npm run start:dev

# production mode
$ npm run start:prod
```

## Run tests

```bash
# unit tests
$ npm run test

# e2e tests
$ npm run test:e2e

# test coverage
$ npm run test:cov
```

## Deployment

When you're ready to deploy your NestJS application to production, there are some key steps you can take to ensure it runs as efficiently as possible. Check out the [deployment documentation](https://docs.nestjs.com/deployment) for more information.

If you are looking for a cloud-based platform to deploy your NestJS application, check out [Mau](https://mau.nestjs.com), our official platform for deploying NestJS applications on AWS. Mau makes deployment straightforward and fast, requiring just a few simple steps:

```bash
$ npm install -g @nestjs/mau
$ mau deploy
```

With Mau, you can deploy your application in just a few clicks, allowing you to focus on building features rather than managing infrastructure.

## Seeding inicial (primer usuario)

Este proyecto crea automáticamente un empleado por defecto al primer arranque si la tabla `empleados` está vacía. Esto es útil cuando despliegas en un VPS con Docker y la base de datos comienza sin datos.

- Control de encendido: `SEED_ON_BOOT` (por defecto true en production, false en test)
- Variables por defecto del usuario administrador:
  - `SEED_ADMIN_NAME` (por defecto: `Administrador`)
  - `SEED_ADMIN_EMAIL` (por defecto: `admin@demo.com`)
  - `SEED_ADMIN_PASSWORD` (por defecto: `admin123`)

Ejemplo de `.env` (ver también `.env.example`):

```
DB_HOST=localhost
DB_PORT=5432
DB_USERNAME=postgres
DB_PASSWORD=postgres
DB_NAME=heladeria
NODE_ENV=production
SEED_ON_BOOT=true
SEED_ADMIN_NAME=Administrador
SEED_ADMIN_EMAIL=admin@demo.com
SEED_ADMIN_PASSWORD=admin123
```

Notas:
- La contraseña se encripta automáticamente mediante los hooks `@BeforeInsert/@BeforeUpdate` del entity `Empleado`.
- Si ya existen empleados, el seed no realiza ninguna acción.
- Puedes desactivar el seed estableciendo `SEED_ON_BOOT=false`.

### Docker

Para pasar variables de entorno en Docker, crea un archivo `.env` y móntalo o usa `--env-file` al ejecutar la imagen. Por ejemplo:

```bash
docker run --env-file .env -p 3000:3000 tu-imagen
```
